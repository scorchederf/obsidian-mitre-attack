## Description

Adversaries may use ClickOnce applications (.appref-ms and .application files) to proxy execution of code through a trusted Windows utility.[^8]  ClickOnce is a deployment that enables a user to create self-updating Windows-based .NET applications (i.e, .XBAP, .EXE, or .DLL) that install and run from a file share or web page with minimal user interaction. The application launches as a child process of DFSVC.EXE, which is responsible for installing, launching, and updating the application.[^5] <br><br>Because ClickOnce applications receive only limited permissions, they do not require administrative permissions to install.[^1]  As such, adversaries may abuse ClickOnce to proxy execution of malicious code without needing to escalate privileges.<br><br>ClickOnce may be abused in a number of ways. For example, an adversary may rely on [[kb/mitre/attack/techniques/T1204-User_Execution\|User Execution]]. When a user visits a malicious website, the .NET malware is disguised as legitimate software and a ClickOnce popup is displayed for installation.[^4] <br><br>Adversaries may also abuse ClickOnce to execute malware via a [[kb/mitre/attack/techniques/T1218.011-Rundll32\|Rundll32]] script using the command `rundll32.exe dfshim.dll,ShOpenVerbApplication1`.[^2] <br><br>Additionally, an adversary can move the ClickOnce application file to a remote user’s startup folder for continued malicious code deployment (i.e., [[kb/mitre/attack/techniques/T1547.001-Registry_Run_Keys_Startup_Folder\|Registry Run Keys / Startup Folder]]).[^8] [^6] 




## Mitigations
| ID | Name | Description |
| --- | --- | --- |
| [[kb/mitre/attack/mitigations/M1021-Restrict_Web-Based_Content\|M1021]] | Restrict Web-Based Content | Disable ClickOnce installations from the internet using the following registry key: <br>`\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework\Security\TrustManager\PromptingLevel — Internet:Disabled`[^4]  |
| [[kb/mitre/attack/mitigations/M1042-Disable_or_Remove_Feature_or_Program\|M1042]] | Disable or Remove Feature or Program | Disable ClickOnce installations from the internet using the following registry key: <br>`\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework\Security\TrustManager\PromptingLevel — Internet:Disabled`[^4] [^3] <br><br>ClickOnce may not be necessary within an environment and should be disabled if not being used. |
| [[kb/mitre/attack/mitigations/M1045-Code_Signing\|M1045]] | Code Signing | Enforce binary and application integrity with digital signature verification to prevent untrusted code from executing.[^7]  |
