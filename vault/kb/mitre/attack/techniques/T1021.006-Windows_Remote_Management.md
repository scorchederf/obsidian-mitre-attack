## Description

Adversaries may use [[kb/mitre/attack/techniques/T1078-Valid_Accounts\|Valid Accounts]] to interact with remote systems using Windows Remote Management (WinRM). The adversary may then perform actions as the logged-on user.<br><br>WinRM is the name of both a Windows service and a protocol that allows a user to interact with a remote system (e.g., run an executable, modify the Registry, modify services).[^1]  It may be called with the `winrm` command or by any number of programs such as PowerShell.[^2]  WinRM  can be used as a method of remotely interacting with [[kb/mitre/attack/techniques/T1047-Windows_Management_Instrumentation\|Windows Management Instrumentation]].[^7]
