## Description

Adversaries may abuse SyncAppvPublishingServer.vbs to proxy execution of malicious [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] commands. SyncAppvPublishingServer.vbs is a Visual Basic script associated with how Windows virtualizes applications (Microsoft Application Virtualization, or App-V).[^3]  For example, Windows may render Win32 applications to users as virtual applications, allowing users to launch and interact with them as if they were installed locally.[^1] [^7] <br>    <br>The SyncAppvPublishingServer.vbs script is legitimate, may be signed by Microsoft, and is commonly executed from `\System32` through the command line via `wscript.exe`.[^2] [^5] <br><br>Adversaries may abuse SyncAppvPublishingServer.vbs to bypass [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] execution restrictions and evade defensive counter measures by "living off the land."[^4] [^2]  Proxying execution may function as a trusted/signed alternative to directly invoking `powershell.exe`.[^6] <br><br>For example,  [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] commands may be invoked using:[^5] <br><br>`SyncAppvPublishingServer.vbs "n; {PowerShell}"`
