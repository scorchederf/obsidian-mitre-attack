## Description

Adversaries may attempt to access credentials and other sensitive information by abusing a Windows Domain Controller's application programming interface (API)[^4]  [^2]  [^8]  [^5]  to simulate the replication process from a remote domain controller using a technique called DCSync.<br><br>Members of the Administrators, Domain Admins, and Enterprise Admin groups or computer accounts on the domain controller are able to run DCSync to pull password data[^6]  from Active Directory, which may include current and historical hashes of potentially useful accounts such as KRBTGT and Administrators. The hashes can then in turn be used to create a [[kb/mitre/attack/techniques/T1558.001-Golden_Ticket\|Golden Ticket]] for use in [[kb/mitre/attack/techniques/T1550.003-Pass_the_Ticket\|Pass the Ticket]][^17]  or change an account's password as noted in [[kb/mitre/attack/techniques/T1098-Account_Manipulation\|Account Manipulation]].[^11] <br><br>DCSync functionality has been included in the "lsadump" module in [[kb/mitre/attack/software/S0002-Mimikatz\|Mimikatz]].[^3]  Lsadump also includes NetSync, which performs DCSync over a legacy replication protocol.[^7]

## Mitigations
| ID | Name | Description |
| --- | --- | --- |
| [[kb/mitre/attack/mitigations/M1015-Active_Directory_Configuration\|M1015]] | Active Directory Configuration | Manage the access control list for "Replicating Directory Changes" and other permissions associated with domain controller replication.[^6] [^10]  |
| [[kb/mitre/attack/mitigations/M1026-Privileged_Account_Management\|M1026]] | Privileged Account Management | Do not put user or admin domain accounts in the local administrator groups across systems unless they are tightly controlled, as this is often equivalent to having a local administrator account with the same password on all systems. Follow best practices for design and administration of an enterprise network to limit privileged account use across administrative tiers. |
| [[kb/mitre/attack/mitigations/M1027-Password_Policies\|M1027]] | Password Policies | Ensure that local administrator accounts have complex, unique passwords across all systems on the network. |
