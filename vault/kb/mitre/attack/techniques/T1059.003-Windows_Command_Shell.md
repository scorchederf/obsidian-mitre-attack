## Description

Adversaries may abuse the Windows command shell for execution. The Windows command shell ([[kb/mitre/attack/software/S0106-cmd\|cmd]]) is the primary command prompt on Windows systems. The Windows command prompt can be used to control almost any aspect of a system, with various permission levels required for different subsets of commands. The command prompt can be invoked remotely via [[kb/mitre/attack/techniques/T1021-Remote_Services\|Remote Services]] such as [[kb/mitre/attack/techniques/T1021.004-SSH\|SSH]].[^2] <br><br>Batch files (ex: .bat or .cmd) also provide the shell with a list of sequential commands to run, as well as normal scripting operations such as conditionals and loops. Common uses of batch files include long or repetitive tasks, or the need to run the same set of commands on multiple systems.<br><br>Adversaries may leverage [[kb/mitre/attack/software/S0106-cmd\|cmd]] to execute various commands and payloads. Common uses include [[kb/mitre/attack/software/S0106-cmd\|cmd]] to execute a single command, or abusing [[kb/mitre/attack/software/S0106-cmd\|cmd]] interactively with input and output forwarded over a command and control channel.
