## Description

Adversaries may log user keystrokes to intercept credentials as the user types them. Keylogging is likely to be used to acquire credentials for new access opportunities when [[kb/mitre/attack/techniques/T1003-OS_Credential_Dumping\|OS Credential Dumping]] efforts are not effective, and may require an adversary to intercept keystrokes on a system for a substantial period of time before credentials can be successfully captured. In order to increase the likelihood of capturing credentials quickly, an adversary may also perform actions such as clearing browser cookies to force users to reauthenticate to systems.[^16] <br><br>Keylogging is the most prevalent type of input capture, with many different ways of intercepting keystrokes.[^2]  Some methods include:<br><br>* Hooking API callbacks used for processing keystrokes. Unlike [[kb/mitre/attack/techniques/T1056.004-Credential_API_Hooking\|Credential API Hooking]], this focuses solely on API functions intended for processing keystroke data.<br>* Reading raw keystroke data from the hardware buffer.<br>* Windows Registry modifications.<br>* Custom drivers.<br>* [[kb/mitre/attack/techniques/T1601-Modify_System_Image\|Modify System Image]] may provide adversaries with hooks into the operating system of network devices to read raw keystrokes for login sessions.[^9]
