## Description

Adversaries may mimic common operating system GUI components to prompt users for credentials with a seemingly legitimate prompt. When programs are executed that need additional privileges than are present in the current user context, it is common for the operating system to prompt the user for proper credentials to authorize the elevated privileges for the task (ex: [[kb/mitre/attack/techniques/T1548.002-Bypass_User_Account_Control\|Bypass User Account Control]]).<br><br>Adversaries may mimic this functionality to prompt users for credentials with a seemingly legitimate prompt for a number of reasons that mimic normal usage, such as a fake installer requiring additional access or a fake malware removal suite.[^3]  This type of prompt can be used to collect credentials via various languages such as [[kb/mitre/attack/techniques/T1059.002-AppleScript\|AppleScript]][^5] [^2] [^1]  and [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]].[^5] [^6] [^1]  On Linux systems adversaries may launch dialog boxes prompting users for credentials from malicious shell scripts or the command line (i.e. [[kb/mitre/attack/techniques/T1059.004-Unix_Shell\|Unix Shell]]).[^1] <br><br>Adversaries may also mimic common software authentication requests, such as those from browsers or email clients. This may also be paired with user activity monitoring (i.e., [[kb/mitre/attack/techniques/T1217-Browser_Information_Discovery\|Browser Information Discovery]] and/or [[kb/mitre/attack/techniques/T1010-Application_Window_Discovery\|Application Window Discovery]]) to spoof prompts when users are naturally accessing sensitive sites/data.
