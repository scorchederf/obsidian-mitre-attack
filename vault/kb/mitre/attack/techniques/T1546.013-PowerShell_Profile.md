## Description

Adversaries may gain persistence and elevate privileges by executing malicious content triggered by PowerShell profiles. A PowerShell profile  (`profile.ps1`) is a script that runs when [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] starts and can be used as a logon script to customize user environments.<br><br>[[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] supports several profiles depending on the user or host program. For example, there can be different profiles for [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] host programs such as the PowerShell console, PowerShell ISE or Visual Studio Code. An administrator can also configure a profile that applies to all users and host programs on the local computer. [^2]  <br><br>Adversaries may modify these profiles to include arbitrary commands, functions, modules, and/or [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] drives to gain persistence. Every time a user opens a [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] session the modified script will be executed unless the `-NoProfile` flag is used when it is launched. [^1]  <br><br>An adversary may also be able to escalate privileges if a script in a PowerShell profile is loaded and executed by an account with higher privileges, such as a domain administrator. [^5] 




## Mitigations
| ID | Name | Description |
| --- | --- | --- |
| [[kb/mitre/attack/mitigations/M1022-Restrict_File_and_Directory_Permissions\|M1022]] | Restrict File and Directory Permissions | Making PowerShell profiles immutable and only changeable by certain administrators will limit the ability for adversaries to easily create user level persistence. |
| [[kb/mitre/attack/mitigations/M1045-Code_Signing\|M1045]] | Code Signing | Enforce execution of only signed PowerShell scripts. Sign profiles to avoid them from being modified. |
| [[kb/mitre/attack/mitigations/M1054-Software_Configuration\|M1054]] | Software Configuration | Avoid PowerShell profiles if not needed. Use the -No Profile flag with when executing PowerShell scripts remotely to prevent local profiles and scripts from being executed. |
