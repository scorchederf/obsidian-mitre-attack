## Description

Adversaries may search for private key certificate files on compromised systems for insecurely stored credentials. Private cryptographic keys and certificates are used for authentication, encryption/decryption, and digital signatures.[^5]  Common key and certificate file extensions include: .key, .pgp, .gpg, .ppk., .p12, .pem, .pfx, .cer, .p7b, .asc. <br><br>Adversaries may also look in common key directories, such as `~/.ssh` for SSH keys on * nix-based systems or `C:&#92;Users&#92;(username)&#92;.ssh&#92;` on Windows. Adversary tools may also search compromised systems for file extensions relating to cryptographic keys and certificates.[^9] [^2] <br><br>When a device is registered to Entra ID, a device key and a transport key are generated and used to verify the device’s identity.[^4]  An adversary with access to the device may be able to export the keys in order to impersonate the device.[^3] <br><br>On network devices, private keys may be exported via [[kb/mitre/attack/techniques/T1059.008-Network_Device_CLI\|Network Device CLI]] commands such as `crypto pki export`.[^1]  <br><br>Some private keys require a password or passphrase for operation, so an adversary may also use [[kb/mitre/attack/techniques/T1056-Input_Capture\|Input Capture]] for keylogging or attempt to [[kb/mitre/attack/techniques/T1110-Brute_Force\|Brute Force]] the passphrase off-line. These private keys can be used to authenticate to [[kb/mitre/attack/techniques/T1021-Remote_Services\|Remote Services]] like SSH or for use in decrypting other collected files such as email.

| [[kb/mitre/attack/software/S0363-Empire\|S0363]] | Empire | [[kb/mitre/attack/software/S0363-Empire\|Empire]] can use modules like `Invoke-SessionGopher` to extract private key and session information.[^6]  |

## Mitigations
| ID | Name | Description |
| --- | --- | --- |
| [[kb/mitre/attack/mitigations/M1022-Restrict_File_and_Directory_Permissions\|M1022]] | Restrict File and Directory Permissions | Ensure permissions are properly set on folders containing sensitive private keys to prevent unintended access. Additionally, on Cisco devices, set the `nonexportable` flag during RSA key pair generation.[^1]  |
| [[kb/mitre/attack/mitigations/M1027-Password_Policies\|M1027]] | Password Policies | Use strong passphrases for private keys to make cracking difficult. |
| [[kb/mitre/attack/mitigations/M1041-Encrypt_Sensitive_Information\|M1041]] | Encrypt Sensitive Information | When possible, store keys on separate cryptographic hardware instead of on the local system. For example, on Windows systems use a TPM to secure keys and other sensitive credential material.[^4]  |
| [[kb/mitre/attack/mitigations/M1047-Audit\|M1047]] | Audit | Ensure only authorized keys are allowed access to critical resources and audit access lists regularly. |
