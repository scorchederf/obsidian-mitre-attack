## Description

Adversaries may modify the lifecycle policies of a cloud storage bucket to destroy all objects stored within.  <br><br>Cloud storage buckets often allow users to set lifecycle policies to automate the migration, archival, or deletion of objects after a set period of time.[^1] [^5] [^3]  If a threat actor has sufficient permissions to modify these policies, they may be able to delete all objects at once. <br><br>For example, in AWS environments, an adversary with the `PutLifecycleConfiguration` permission may use the `PutBucketLifecycle` API call to apply a lifecycle policy to an S3 bucket that deletes all objects in the bucket after one day.[^4] [^2]  In addition to destroying data for purposes of extortion and [[kb/mitre/attack/techniques/T1657-Financial_Theft\|Financial Theft]], adversaries may also perform this action on buckets storing cloud logs for [[kb/mitre/attack/techniques/T1070-Indicator_Removal\|Indicator Removal]].[^6] 




## Mitigations
| ID | Name | Description |
| --- | --- | --- |
| [[kb/mitre/attack/mitigations/M1018-User_Account_Management\|M1018]] | User Account Management | In cloud environments, limit permissions to modify cloud bucket lifecycle policies (e.g., `PutLifecycleConfiguration` in AWS) to only those accounts that require it. In AWS environments, consider using Service Control policies to limit the use of the `PutBucketLifecycle` API call.  |
| [[kb/mitre/attack/mitigations/M1053-Data_Backup\|M1053]] | Data Backup | Consider implementing IT disaster recovery plans that contain procedures for taking regular data backups that can be used to restore organizational data.[^7]  Ensure backups are stored off system and protected from common methods adversaries may use to gain access and destroy the backups to prevent recovery. |
