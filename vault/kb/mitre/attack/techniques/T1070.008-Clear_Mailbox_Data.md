## Description

Adversaries may modify mail and mail application data to remove evidence of their activity. Email applications allow users and other programs to export and delete mailbox data via command line tools or use of APIs. Mail application data can be emails, email metadata, or logs generated by the application or operating system, such as export requests. <br><br>Adversaries may manipulate emails and mailbox data to remove logs, artifacts, and metadata, such as evidence of [[kb/mitre/attack/techniques/T1566-Phishing\|Phishing]]/[[kb/mitre/attack/techniques/T1534-Internal_Spearphishing\|Internal Spearphishing]], [[kb/mitre/attack/techniques/T1114-Email_Collection\|Email Collection]], [[kb/mitre/attack/techniques/T1071.003-Mail_Protocols\|Mail Protocols]] for command and control, or email-based exfiltration such as [[kb/mitre/attack/techniques/T1048-Exfiltration_Over_Alternative_Protocol\|Exfiltration Over Alternative Protocol]]. For example, to remove evidence on Exchange servers adversaries have used the `ExchangePowerShell` [[kb/mitre/attack/techniques/T1059.001-PowerShell\|PowerShell]] module, including `Remove-MailboxExportRequest` to remove evidence of mailbox exports.[^5] [^4]  On Linux and macOS, adversaries may also delete emails through a command line utility called `mail`  or use [[kb/mitre/attack/techniques/T1059.002-AppleScript\|AppleScript]] to interact with APIs on macOS.[^1] [^2] <br><br>Adversaries may also remove emails and metadata/headers indicative of spam or suspicious activity (for example, through the use of organization-wide transport rules) to reduce the likelihood of malicious emails being detected by security products.[^3] 




## Mitigations
| ID | Name | Description |
| --- | --- | --- |
| [[kb/mitre/attack/mitigations/M1022-Restrict_File_and_Directory_Permissions\|M1022]] | Restrict File and Directory Permissions | Protect generated event files that are stored locally with proper permissions and authentication and limit opportunities for adversaries to increase privileges by preventing Privilege Escalation opportunities.  |
| [[kb/mitre/attack/mitigations/M1029-Remote_Data_Storage\|M1029]] | Remote Data Storage | Automatically forward mail data and events to a log server or data repository to prevent conditions in which the adversary can locate and manipulate data on the local system. When possible, minimize time delay on event reporting to avoid prolonged storage on the local system.  |
| [[kb/mitre/attack/mitigations/M1047-Audit\|M1047]] | Audit | In an Exchange environment, Administrators can use `Get-TransportRule` / `Remove-TransportRule` to discover and remove potentially malicious transport rules.[^6]  |
